# 校园二手交易平台完整版建设总方案

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将当前 HarmonyOS 校园电商原型升级为前后端分离、数据库上云、可部署演示的校园二手交易平台完整版。

**Architecture:** 客户端继续使用 HarmonyOS ArkTS/ArkUI 负责页面与交互，Node.js 后端统一承接认证、商品、订单、聊天、公告、后台管理等业务接口，MySQL 负责持久化。Nginx 作为反向代理与静态资源入口，PM2 负责 Node 服务守护。

**Tech Stack:** HarmonyOS NEXT, ArkTS, ArkUI, Node.js 20, Express, MySQL 8.0, Nginx, PM2, JWT, bcrypt, multer

### Task 1: Confirm product boundary

**Files:**

- [ ] **Step 1: Freeze the product positioning**

System positioning:
- Campus second-hand trading platform
- Users are unified identities; there is no separate merchant onboarding system
- The same user can act as both buyer and seller
- Simulated payment is retained
- Campus identity verification, map navigation, real third-party payment, and enterprise-grade high-concurrency capabilities are excluded from this implementation

- [ ] **Step 2: Freeze the required business modules**

Required modules:
- Authentication and registration
- Product publish/list/detail/search
- Order create/pay/arrive/complete/review
- Buyer-seller chat
- Announcement center
- Admin dashboard and governance
- Cloud database persistence

- [ ] **Step 3: Freeze the deployment model**

Deployment model:
- HarmonyOS client runs on device/emulator
- Node.js backend runs on Tencent Cloud server
- MySQL runs on the same server
- Nginx reverse proxies backend and static uploads

## Acceptance Baseline

The implementation is considered complete only when:
- Client login and session restore depend on server data
- Product, order, review, announcement, and chat data are stored in MySQL
- Buyer and seller can communicate inside the platform
- Admin can govern users, products, and announcements
- Backend survives server restart through PM2
- Nginx can proxy requests from public IP
- HarmonyOS client can complete at least one full trading flow against the cloud backend
