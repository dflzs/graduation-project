# 新增商品审核流路由说明

当前 `backend_code` 目录里只有 controller，没有 router 文件。  
如果要把这轮“发布 -> 审核 -> 上架 / 驳回”真正同步到云端，请在云端后端路由里补上下面这些映射。

## 商品路由

```js
const express = require('express');
const router = express.Router();
const authMiddleware = require('../middleware/auth.middleware');
const {
  createProduct,
  listProducts,
  listMyProducts,
  getProductDetail
} = require('../controllers/product.controller');

router.get('/', listProducts);
router.get('/mine', authMiddleware, listMyProducts);
router.get('/:id', getProductDetail);
router.post('/', authMiddleware, createProduct);

module.exports = router;
```

## 后台商品管理路由

```js
const express = require('express');
const router = express.Router();
const authMiddleware = require('../middleware/auth.middleware');
const {
  dashboard,
  listUsers,
  updateUserStatus,
  listProducts,
  approveProduct,
  rejectProduct,
  forceOffShelfProduct
} = require('../controllers/admin.controller');

router.get('/dashboard', authMiddleware, dashboard);
router.get('/users', authMiddleware, listUsers);
router.put('/users/:id/status', authMiddleware, updateUserStatus);

router.get('/products', authMiddleware, listProducts);
router.put('/products/:id/approve', authMiddleware, approveProduct);
router.put('/products/:id/reject', authMiddleware, rejectProduct);
router.put('/products/:id/off-shelf', authMiddleware, forceOffShelfProduct);

module.exports = router;
```

## 这轮上线后的预期行为

- 用户发布商品后，数据库状态默认是 `pending_review`
- 首页公共商品流默认只返回 `on_sale`
- `GET /api/products/mine` 能看到当前登录用户自己的全部商品，包括 `pending_review`
- 管理员可以通过：
  - `PUT /api/admin/products/:id/approve`
  - `PUT /api/admin/products/:id/reject`
  - `PUT /api/admin/products/:id/off-shelf`
  来处理商品状态
