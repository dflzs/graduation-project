# 治理与申诉后端接线说明

当前工作区里只有治理相关的 `controller / routes / sql` 文件，没有服务器实际运行中的 `src/routes/index.js`，因此本地仓库无法直接把新路由挂到真实启动入口。

在云端后端项目中，需要把以下 3 个路由文件同步到：

- `src/routes/moderation.routes.js`
- `src/routes/appeal.routes.js`
- `src/routes/governance.routes.js`

然后在 `src/routes/index.js` 中补上：

```js
const moderationRoutes = require('./moderation.routes');
const appealRoutes = require('./appeal.routes');
const governanceRoutes = require('./governance.routes');
```

并在现有 `router.use(...)` 区域增加：

```js
router.use(moderationRoutes);
router.use(appealRoutes);
router.use(governanceRoutes);
```

说明：

- 这 3 个路由文件内部已经自行声明了完整路径前缀，例如 `/moderation/records`、`/appeals`、`/admin/governance/actions`。
- 因此这里不再额外挂二级前缀，统一直接 `router.use(...)` 即可。
- 外层应用仍保持 `app.use('/api', routes)`，最终接口路径分别会落成：
  - `/api/moderation/records`
  - `/api/appeals`
  - `/api/admin/moderation/records`
  - `/api/admin/appeals`
  - `/api/governance/users/:id/summary`
  - `/api/admin/governance/actions`

数据库方面，需要在目标库执行：

- `backend_code/governance-trust.sql`

该 SQL 会完成：

- `users.status` 放宽到 `VARCHAR(32)`，支持 `active / restricted / muted / suspended / banned`
- 创建 `moderation_records`
- 创建 `appeals`
- 创建 `governance_actions`
- 补齐治理域所需索引
